# Project Rules

These rules define how Project Genesis is developed. They exist to ensure consistency, high-quality decisions, and long-term thinking.

1. Research before decisions.
2. Decisions before development.
3. Every feature must solve a real problem or create a meaningful positive moment.
4. User well-being always comes before engagement.
5. Simplicity is preferred over feature quantity.
6. Privacy is a default, not an option.
7. Every important decision must be documented.
8. The repository is the Single Source of Truth.
9. Quality is more important than speed.
10. Challenge assumptions with evidence.

These rules may evolve, but only after careful discussion and evidence.
